# PV参数验证测试设计方法

## 验证维度（按序检查）

### 1. 必填性验证
- **空值测试**：`null`、`undefined`、空字符串、纯空格
- **缺失测试**：完全省略该参数
- **默认值测试**：参数缺失时系统默认值

### 2. 数据类型验证
- **类型错误**：数字传字符串、布尔传数字等
  - **排除**：当接口文档已明确参数为强类型（int/long/boolean/double 等）时，不生成"传入字符串"的用例。框架层类型转换（如 Spring 的 TypeMismatchException）不属于业务逻辑测试范畴。仅对 String 类型参数的业务格式约束（如手机号、邮箱）生成类型/格式验证用例。
- **格式验证**：邮箱、手机号、URL、日期格式等
- **编码问题**：特殊字符、Unicode、编码格式

### 3. 长度边界验证（补充BVA）
- **字符串长度**：最小长度、最大长度、超长截断
- **数组长度**：空数组、单元素、最大容量、超限
- **文件大小**：0字节、正常大小、超大文件

### 4. 取值范围验证（补充BVA）
- **数值范围**：负数、零、正数、小数精度
- **枚举值**：非法枚举值、大小写敏感
- **业务约束**：身份证号校验、金额限制等

### 5. 参数依赖关系验证
- **互斥参数**：两个互斥参数同时传入（如"手机号登录"和"邮箱登录"的参数同时存在）
- **条件必填**：当参数 A 存在时，参数 B 必填（如"选择快递配送时，收货地址必填"）
- **依赖缺失**：参数 B 依赖参数 A，但 A 未传入（如"传了 endDate 但没传 startDate"）
- **组合边界**：参数组合产生的隐式边界（如分页参数 page × size 乘积过大、日期范围 startDate > endDate）
- **联动约束**：参数值之间的业务联动（如"类型为个人时，公司名称应忽略或报错"）

## 生成规则

### 触发条件
- API接口的请求参数
- 函数/方法的输入参数  
- 表单提交的字段
- 配置项的设置参数

### 用例生成模板
- PV-必填性: 验证[参数名]必填性 - [测试场景]
- PV-数据类型: 验证[参数名]数据类型 - [非法类型]
- PV-长度边界: 验证[参数名]长度边界 - [边界值]
- PV-取值范围: 验证[参数名]取值范围 - [非法值]
- PV-参数依赖: 验证[参数A]与[参数B]依赖关系 - [违反场景]

### 优先级分配

基础优先级按验证维度分配，再根据业务风险上调：

| 验证维度 | 基础优先级 | 说明 |
|---------|-----------|------|
| 必填性验证 | **P1** | 必填参数缺失通常导致功能不可用 |
| 关键业务参数的数据类型验证 | **P1** | 类型错误可能导致异常或数据损坏 |
| 长度边界验证 | **P2** | 超长输入可能导致截断或存储异常 |
| 非关键参数的数据类型验证 | **P2** | 影响范围有限 |
| 取值范围验证 | **P2** | 非法取值可能影响业务逻辑 |
| 格式验证 | **P2** | 格式错误通常有明确的校验提示 |
| 参数依赖关系验证 | **P2** | 参数组合错误可能导致逻辑异常 |

**风险修正规则**（在基础优先级上向上调整）：

| 风险特征 | 修正动作 | 示例 |
|---------|---------|------|
| 可能导致资损（金额、数量、价格相关） | 上调至 **P1** | 金额为负数、数量为 0、价格精度溢出 |
| 可能导致越权或数据泄露 | 上调至 **P1** | 枚举值越权（如 role 传 admin）、状态值篡改 |
| 可能导致数据不一致 | 上调至 **P1** | 分页参数 page*size 乘积过大导致 OOM |
| 仅影响展示、不影响数据 | 可降至 **P3** | 备注字段格式、非必填字段的长度边界 |
